== Описание ==

короткие пошаговые сражения на маленькой карте. различные типы войск и
неосведомленность о выборе противка. сражения могут ранжироваться по стоимости
войск в каждом отряде. для такой игры нужна песочница для отладки тактик. и эта
песочница будет очень важной частью игры. программировать нужно будет функцию
step для каждого юнита и стратегической функции step, результатами работы
которой могут пользоваться юниты. так например стратегическая функция может
выбирать приоритетные цели и назначать их определенным юнитам. так же
стратегическая функция может менять тактические функции юнитам.

типы войск:
- солдат (жизни 100, атака 20, дальность 1, скорость 5)
- лучник (жизни  70, атака 10, дальность 5, скорость 7) не может стрелять,
если рядом вражеский солдат.

примером поведения может быть:
- подойти к ближайшему противнику и бить его до чьей-нибудь смерти.
- выбрать цель можно анализируя состав войск противника.
- лучник может чередовать отступление от солдата и стрельбу в него.

== Описание функционала ==

Пользователи (рейтинг Эло), стратегические функции, тактические функции юнитов,
отряды (набор юнитов + тактические функции + стратегическая функция), повторы
сражений, выставленные для сражений отряды. песочница.
        Рейтинг Эло должен быть у пользователя а не у отряда, уровень любого
        отряда примерно отражает уровень пользователя. Так же как в шахматах
        игрок может использовать различные стратегии и тактики, поэтому ничего
        страшного в том, чтобы к разным по силе отрядам относился один рейтинг
        пользователя. Но у отрядов тоже должен быть свой рейтинг, чтобы
        пользователь оценить успешность той или иной стратегии. Точнее рейтинги
        должны быть у каждой версии отряда выставленного на бой.

Главная страница может представлять собо поле песочницы готовое для
программирования + авторизация. в песочнице должно быть поле, редактор кода,
кнопки управления (run, pause, step).


Сражения между собственными отрядами никак не должны влиять на публичные
показатели пользователя. И наоборот, сражения с отрядами других пользователей
всегда должны быть публичными (не должно быть возможности в тихоря настроить
свой отряд против конкретного отряда противника).

Вызов на бой должен быть всегда публичный, чтобы любой желающий мог откликнуться
на него. Чтобы повторы матчей представляли интерес, должны храниться версии
исходников на момент матча. Получается, что выставление отряда на бой - это
по-сути commit)

== Сценарии ==

=== программирование ===
 - авторизоваться
 - выбать/создать отряд
 - выбрать противника (default, либо другой свой отряд)
 - программировать/отлаживать свой отряд

=== матч ===
 - авторизоваться
 - выбрать отряд
 - выставить на бой/выбрать выставленный отряд для боя
 - посмотреть повтор

=== история ===

 - должны быть доступны истории всех сражений для каждого отряда вместе с
   исходниками.
